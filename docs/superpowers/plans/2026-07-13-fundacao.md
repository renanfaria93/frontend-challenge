# Fundação Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Criar a base do frontend (scaffolding Vite+React+TS+Tailwind+shadcn, tipos de entidade, camada de API genérica, biblioteca de componentes reutilizáveis, layout/rotas) e o contrato de API (`openapi.yaml` + Swagger UI estático), para que os 5 planos seguintes (CRUD Alunos, CRUD Professores, CRUD Disciplinas, Dashboard, Docker/entrega) só precisem consumir essa base.

**Architecture:** Vite + React + TypeScript com Tailwind v4 (via `@tailwindcss/vite`, instalado pelo `shadcn init`) e shadcn/ui para os primitivos visuais. Uma factory genérica `createEntityHooks<TEntity, TInput>(endpoint, entityLabel)` centraliza `useList/useCreate/useUpdate/useDelete` (TanStack Query + Axios + toasts `sonner`) para as 3 entidades. Componentes de UI repetidos (tabela, modais, estados vazio/erro/loading, etc.) vivem em `components/shared/` e são consumidos por todas as páginas de CRUD construídas nos planos seguintes. `AppShell` fornece layout com sidebar fixa em desktop e off-canvas (`Sheet`) abaixo de 1024px.

**Tech Stack:** React + Vite + TypeScript, React Router, Tailwind CSS v4, shadcn/ui, Lucide React, React Hook Form + Zod, Axios, TanStack Query, sonner.

## Global Constraints

- Stack obrigatória: React + Vite + TypeScript, React Router, Tailwind CSS, shadcn/ui, Lucide React, React Hook Form + Zod, Axios, TanStack Query. Sem GraphQL.
- Nunca usar dados mockados, faker, JSON local, fixtures ou MSW/json-server — toda informação vem da API real via `VITE_API_URL`.
- CRUD: sempre via a factory genérica `createEntityHooks<T>(endpoint)` — nunca duplicar lógica de fetch por entidade.
- Criar/editar: sempre via `FormModal`. Excluir: sempre via `ConfirmDialog`. Nunca criar página separada de criação/edição.
- Antes de criar qualquer componente novo, verificar se já existe em `components/shared/`. shadcn/ui primeiro, sempre, antes de construir algo do zero.
- Domínio em português nos campos de entidade e labels de UI (`nome`, `matricula`, `cargaHoraria`, `professorId`, `alunosIds`); nomes técnicos (funções, hooks, componentes) em inglês.
- Toasts via `sonner` em toda mutation (sucesso e erro); botões desabilitados durante submissão.
- Tipagem forte, sem `any`. Tipos de entidade centralizados em `src/types/`.
- Cores: base neutra + indigo (`#4f46e5` claro / `#6366f1` escuro) como destaque.
- Qualquer tarefa visual (tela nova, componente de UI, layout/espaçamento/cor/tipografia) **deve invocar a skill `frontend-design` antes de implementar**, mesmo em ajustes pequenos — exigência do `CLAUDE.md` deste repositório.
- Não existe suíte de testes automatizados neste projeto (fora de escopo). Verificação é manual: `npm run build` (checagem de tipos) e checagem visual via `npm run dev` + navegador.
- Erros de API sempre no formato `{ "message": string }`.

### Task 7: Factory genérica de hooks CRUD (`entityFactory`)

**Files:**
- Create: `src/api/entityFactory.ts`

**Interfaces:**
- Consumes: `apiClient`, `getApiErrorMessage` (`@/api/client`, Task 6)
- Produces: `createEntityHooks<TEntity extends { id: number }, TInput>(endpoint: string, entityLabel: string)` retornando `{ useList, useCreate, useUpdate, useDelete, queryKey }`, onde:
  - `useList(): UseQueryResult<TEntity[]>`
  - `useCreate(): UseMutationResult<TEntity, unknown, TInput>`
  - `useUpdate(): UseMutationResult<TEntity, unknown, { id: number; input: TInput }>`
  - `useDelete(): UseMutationResult<void, unknown, number>`

  Usado por `features/alunos/useAlunos.ts`, `features/professores/useProfessores.ts` e `features/disciplinas/useDisciplinas.ts` nos planos seguintes.

- [ ] **Step 1: Criar `src/api/entityFactory.ts`**

```ts
import { useMutation, useQuery, useQueryClient } from "@tanstack/react-query";
import { toast } from "sonner";
import { apiClient, getApiErrorMessage } from "./client";

export function createEntityHooks<TEntity extends { id: number }, TInput>(
  endpoint: string,
  entityLabel: string,
) {
  const queryKey = [endpoint];

  function useList() {
    return useQuery({
      queryKey,
      queryFn: async () => {
        const { data } = await apiClient.get<TEntity[]>(`/${endpoint}`);
        return data;
      },
    });
  }

  function useCreate() {
    const queryClient = useQueryClient();
    return useMutation({
      mutationFn: async (input: TInput) => {
        const { data } = await apiClient.post<TEntity>(`/${endpoint}`, input);
        return data;
      },
      onSuccess: () => {
        queryClient.invalidateQueries({ queryKey });
        toast.success(`${entityLabel} criado com sucesso.`);
      },
      onError: (error) => {
        toast.error(getApiErrorMessage(error));
      },
    });
  }

  function useUpdate() {
    const queryClient = useQueryClient();
    return useMutation({
      mutationFn: async ({ id, input }: { id: number; input: TInput }) => {
        const { data } = await apiClient.put<TEntity>(`/${endpoint}/${id}`, input);
        return data;
      },
      onSuccess: () => {
        queryClient.invalidateQueries({ queryKey });
        toast.success(`${entityLabel} atualizado com sucesso.`);
      },
      onError: (error) => {
        toast.error(getApiErrorMessage(error));
      },
    });
  }

  function useDelete() {
    const queryClient = useQueryClient();
    return useMutation({
      mutationFn: async (id: number) => {
        await apiClient.delete(`/${endpoint}/${id}`);
      },
      onSuccess: () => {
        queryClient.invalidateQueries({ queryKey });
        toast.success(`${entityLabel} excluído com sucesso.`);
      },
      onError: (error) => {
        toast.error(getApiErrorMessage(error));
      },
    });
  }

  return { useList, useCreate, useUpdate, useDelete, queryKey };
}
```

### Task 13: Componente shared — `DataTable`

**Antes de implementar:** invocar a skill `frontend-design`.

**Files:**
- Create: `src/components/shared/DataTable.tsx`

**Interfaces:**
- Consumes: `Table, TableBody, TableCell, TableHead, TableHeader, TableRow` (`@/components/ui/table`, Task 2), `Button` (`@/components/ui/button`)
- Produces: `interface DataTableColumn<T> { key: string; header: string; render: (item: T) => ReactNode; className?: string }` e `DataTable<T>({ data, columns, getRowId, pageSize }: { data: T[]; columns: DataTableColumn<T>[]; getRowId: (item: T) => number | string; pageSize?: number })`, com paginação client-side (10 itens/página por padrão).

  **Nota para quem consumir este componente:** o estado de página é interno ao `DataTable` e não reresume para 1 quando a prop `data` muda (ex.: ao digitar num filtro). Para evitar ficar preso numa página vazia após filtrar, quem renderizar `<DataTable />` sobre uma lista filtrada deve passar `key={filtroTexto + filtroSelect}` para forçar remontagem quando os filtros mudam. Essa orientação é repetida nos planos de CRUD (Alunos, Professores, Disciplinas).

- [ ] **Step 1: Invocar a skill `frontend-design`**

- [ ] **Step 2: Criar `src/components/shared/DataTable.tsx`**

```tsx
import { useState, type ReactNode } from "react";
import {
  Table,
  TableBody,
  TableCell,
  TableHead,
  TableHeader,
  TableRow,
} from "@/components/ui/table";
import { Button } from "@/components/ui/button";

export interface DataTableColumn<T> {
  key: string;
  header: string;
  render: (item: T) => ReactNode;
  className?: string;
}

interface DataTableProps<T> {
  data: T[];
  columns: DataTableColumn<T>[];
  getRowId: (item: T) => number | string;
  pageSize?: number;
}

export function DataTable<T>({ data, columns, getRowId, pageSize = 10 }: DataTableProps<T>) {
  const [page, setPage] = useState(1);
  const totalPages = Math.max(1, Math.ceil(data.length / pageSize));
  const currentPage = Math.min(page, totalPages);
  const start = (currentPage - 1) * pageSize;
  const pageItems = data.slice(start, start + pageSize);

  return (
    <div className="flex flex-col gap-4">
      <div className="overflow-x-auto rounded-md border">
        <Table>
          <TableHeader>
            <TableRow>
              {columns.map((column) => (
                <TableHead key={column.key} className={column.className}>
                  {column.header}
                </TableHead>
              ))}
            </TableRow>
          </TableHeader>
          <TableBody>
            {pageItems.map((item) => (
              <TableRow key={getRowId(item)}>
                {columns.map((column) => (
                  <TableCell key={column.key} className={column.className}>
                    {column.render(item)}
                  </TableCell>
                ))}
              </TableRow>
            ))}
          </TableBody>
        </Table>
      </div>
      {totalPages > 1 ? (
        <div className="flex items-center justify-between text-sm text-muted-foreground">
          <span>
            Página {currentPage} de {totalPages}
          </span>
          <div className="flex gap-2">
            <Button
              variant="outline"
              size="sm"
              disabled={currentPage === 1}
              onClick={() => setPage((p) => Math.max(1, p - 1))}
            >
              Anterior
            </Button>
            <Button
              variant="outline"
              size="sm"
              disabled={currentPage === totalPages}
              onClick={() => setPage((p) => Math.min(totalPages, p + 1))}
            >
              Próxima
            </Button>
          </div>
        </div>
      ) : null}
    </div>
  );
}
```
